1. Introduction to Node.js

Node.js is an open-source, cross-platform JavaScript runtime environment that allows developers to execute JavaScript code outside of a web browser. It uses Google Chrome's V8 JavaScript engine.

Before Node.js, JavaScript was primarily confined to client-side (frontend) web development. Node.js enabled JavaScript to be used for server-side (backend) development, command-line tools, and desktop applications, creating a full-stack JavaScript ecosystem.

2. How Node.js Works

Node.js operates on a single-threaded, event-driven, non-blocking I/O model, which makes it lightweight and efficient.

  • V8 Engine: It compiles JavaScript directly into machine code, making it very fast.
  • Event Loop: Node.js uses an event loop to handle concurrent operations. Instead of waiting for an I/O operation (like a database query or file read) to complete, it sends the request and continues processing other requests. When the I/O operation finishes, it's added to the event queue.
  • Non-blocking I/O: This means that a process does not have to wait for an I/O operation to complete before continuing. This allows Node.js to handle a large number of concurrent connections efficiently.

3. Installation

Installing Node.js is straightforward. It also comes bundled with npm (Node Package Manager).

Steps:

  1. Go to the official Node.js website: nodejs.org/en/download/
  2. Download the installer for your operating system (Windows, macOS, Linux).
  3. Follow the installation prompts.
  4. Verify installation by opening your terminal/command prompt and typing:

node -v
npm -v

These commands should output the installed versions of Node.js and npm respectively.

4. Your First Node.js Application (HTTP Server)

Let's create a simple "Hello World" HTTP server using Node.js.

1. Create a file named app.js: 


const http = require('http'); // Import the built-in http module

const hostname = '127.0.0.1'; // Localhost
const port = 3000; // Port number to listen on

// Create a server instance
const server = http.createServer((req, res) => {
  // Set the response HTTP header with HTTP status and Content-Type
  res.statusCode = 200; // OK
  res.setHeader('Content-Type', 'text/plain'); // Plain text response

  // Send the response body "Hello World"
  res.end('Hello World from Node.js!\n');
});

// The server listens on the specified port and hostname
server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});

2. Run the application from your terminal: 


node app.js

Now, open your web browser and navigate to http://127.0.0.1:3000/. You should see "Hello World from Node.js!".

5. NPM (Node Package Manager)

npm is the default package manager for Node.js. It's used to install, manage, and share JavaScript packages (libraries, frameworks, tools).

Common NPM Commands:

  • npm init: Initializes a new Node.js project, creating a package.json file.
  • npm install <package-name>: Installs a specific package locally.
  • npm install -g <package-name>: Installs a package globally (for command-line tools).
  • npm install: Installs all dependencies listed in package.json.
  • npm uninstall <package-name>: Uninstalls a package.

# Initialize a new project
npm init -y

# Install the Express.js framework
npm install express

6. Node.js Modules

Node.js has a simple module system that allows you to organize your code into reusable units. There are three types of modules:

  • Built-in Modules: Provided by Node.js (e.g., http, fs for file system, path). You use require() to import them.
  • Local Modules: Your own files. You export functionality using module.exports and import using require().
  • Third-party Modules: Installed via npm (e.g., express, lodash). Imported using require().

Example: Creating a Local Module

my-module.js: 


// my-module.js
function sayHello(name) {
  return `Hello, ${name}!`;
}

module.exports = {
  sayHello: sayHello,
  version: '1.0'
};

app.js: 


// app.js
const myModule = require('./my-module'); // Relative path for local module

console.log(myModule.sayHello('Node.js User')); // Output: Hello, Node.js User!
console.log('Module Version:', myModule.version); // Output: Module Version: 1.0

7. Asynchronous JavaScript in Node.js

Due to its non-blocking nature, asynchronous programming is central to Node.js.

Callbacks:

Functions passed as arguments to other functions, to be executed later.


const fs = require('fs'); // File System module

fs.readFile('example.txt', 'utf8', (err, data) => {
  if (err) {
    console.error('Error reading file:', err);
    return;
  }
  console.log('File content:', data);
});
console.log('This will print before file content (non-blocking)');

Promises (ES6+):

A cleaner way to handle asynchronous operations, representing a value that may be available now, or in the future, or never.


const fs = require('fs').promises; // Use promise-based fs module

fs.readFile('example.txt', 'utf8')
  .then(data => {
    console.log('File content (Promise):', data);
  })
  .catch(err => {
    console.error('Error reading file (Promise):', err);
  });

Async/Await (ES2017+):

Syntactic sugar built on Promises, making asynchronous code look and behave more like synchronous code.


const fs = require('fs').promises;

async function readFileAsync() {
  try {
    const data = await fs.readFile('example.txt', 'utf8');
    console.log('File content (Async/Await):', data);
  } catch (err) {
    console.error('Error reading file (Async/Await):', err);
  }
}
readFileAsync();

8. Why Use Node.js?

Node.js is a popular choice for many types of applications:

  • Real-time Applications: Like chat applications, online gaming, and collaboration tools due to its event-driven architecture.
  • APIs (RESTful Services): Building fast and scalable backend APIs.
  • Microservices: Creating small, independent services.
  • Streaming Data: Efficiently handling large amounts of data in streams.
  • Full-stack JavaScript: Using JavaScript for both frontend (React, Vue, Angular) and backend (Node.js) development.
