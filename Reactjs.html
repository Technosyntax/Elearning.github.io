1. Introduction

Welcome to this HTML-based explanation of React.js! While React itself is a JavaScript library used to build dynamic web applications, this page will break down its core concepts using static HTML and CSS.

React has revolutionized front-end development by making it easier to create complex, interactive user interfaces efficiently.

2. What is React.js?

React.js (often simply called React) is a JavaScript library for building user interfaces. It's not a full-fledged framework, but rather focuses on the "view" layer of an application.

  • Declarative: React makes it painless to create interactive UIs. Design simple views for each state in your application, and React will efficiently update and render just the right components when your data changes.
  • Component-Based: Build encapsulated components that manage their own state, then compose them to make complex UIs.
  • Learn Once, Write Anywhere: Develop new features in React without rewriting existing code. React can also render on the server using Node and power mobile apps using React Native.
  • Virtual DOM: React uses a Virtual DOM, which is a lightweight copy of the actual DOM. React updates the Virtual DOM, compares it to the previous version, and then efficiently updates only the changed parts of the real DOM. This optimizes performance.

3. Components: The Building Blocks

At the heart of React are components. Think of a component as a self-contained, reusable piece of a user interface.

A large UI can be broken down into smaller, manageable components. For example, a Facebook page might have components for a "Post," a "Comment," a "Like Button," and so on.

Conceptual Component Structure:


// Conceptual JavaScript code for a simple React component
// (This would NOT run directly in a browser without a React environment)

function Greeting(props) {
  return (
    <div>
      <h1>Hello, {props.name}!</h1>
      <p>Welcome to React.</p>
    </div>
  );
}

// How it might be used (conceptually):
// <Greeting name="World" />
// <Greeting name="Alice" />
  • Components can be functional (like the example above, using JavaScript functions) or class-based.
  • They receive data through "props" (properties) and can manage their own internal "state."

4. JSX: JavaScript XML

JSX is a syntax extension for JavaScript recommended by React. It allows you to write HTML-like code directly within your JavaScript files.

While it looks like HTML, it's actually JavaScript under the hood. JSX gets "transpiled" (converted) into regular JavaScript function calls that create React elements.

JSX Example:


// This is JSX (looks like HTML inside JavaScript)
const element = <h1 className="my-title">Hello, JSX!</h1>;

// This is what JSX compiles down to (plain JavaScript)
const compiledElement = React.createElement(
  'h1',
  {{ className: 'my-title' }},
  'Hello, JSX!'
);
  • It's faster to write and easier to read than pure JavaScript for UI creation.
  • You can embed JavaScript expressions within JSX using curly braces: {{ expression }}.
  • HTML attributes in JSX are camelCase (e.g., className instead of class).

5. Props and State

These two concepts are fundamental to how React components manage and react to data.

Props (Properties):

  • Read-only data passed from a parent component to a child component.
  • Think of them as arguments passed to a function.
  • Children components should *never* modify the props they receive.

// Conceptual Parent component passing a prop:
// <ChildComponent message="Hello from parent!" />

// Conceptual Child component receiving a prop:
// function ChildComponent(props) {
//   return <p>{{ props.message }}</p>;
// }

State:

  • Data managed within a component that can change over time.
  • When state changes, React re-renders the component (and its children) to reflect the new data.
  • In functional components, state is managed using the useState Hook.

6. Why Use React?

React has gained immense popularity for several reasons:

  • Efficiency: React's Virtual DOM and reconciliation algorithm optimize UI updates, leading to fast applications.
  • Reusability: Components promote reusability, saving development time and ensuring consistency.
  • Maintainability: Component-based architecture makes code easier to understand, debug, and maintain.
  • Large Ecosystem: A vast community and rich ecosystem of tools, libraries, and resources are available.
  • Declarative Approach: Developers describe the desired UI state, and React figures out how to achieve it, reducing imperative manipulation of the DOM.
